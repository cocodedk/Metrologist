# Security Policy

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Report privately through one of these channels:

- **GitHub Security tab** — use the "Report a vulnerability" button on the repository's Security page.
- **Email** — send details to babak@cocode.dk with the subject line `[Metrologist] Security`.

Include a description of the issue, the version or commit affected, and steps to reproduce or a proof-of-concept if you have one. You will receive an acknowledgement within **5 business days** and a resolution or status update within **30 days**.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Older releases | No |

Only the latest stable release receives security fixes. If you are running an older version, update to the current release before reporting.
